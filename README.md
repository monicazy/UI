
简介：主要由Python+selenium+pytest+allure+jenkins构成

----------------------------------------------
主体思想：采用了PageObject设计模式，利用分层思想，将页面功能、测试数据、测试用例分开封装，设计中尽量保持用例的独立性稳定性

----------------------------------------------

更新历史：

2019-1-10：

1. Common是封装的公共类，包含了所有页面公共的基本操作，封装了基本函数 - 执行日志、异常处理、失败截图、文件存放路径
2. 元素定位：元素定位类型和表达式用元组来管理  -  PageLocators层
3. TestResult存放测试报告、测试结果页面的失败截图、执行日志的输出文件
4. 测试数据是将元素定位与用例测试数据分离，定位元素写在PageLocators中，测试数据写在TestDatas中（元素定位数量较少时可以写在同一个文件中）
5. 测试用例主要是由页面对象+测试数据的调用组成，引入pytest的conftest文件用来存放公共的前后置的操作，同时使用pytest的mark功能标记冒烟用例
6. 失败用例的再执行：pytest的rerun功能，能够自动重复执行失败的用例  --reruns 1 --reruns-delay 3
7. 测试报告：--alluredir=TestResult/allure_results
----------------------------------------------
